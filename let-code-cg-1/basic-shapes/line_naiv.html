วาดเส้นตรงจากสมการ

โดยการนำเอาสมการ (1.6) มาคำนวณค่าตำแหน่งของ pixel ทีละ pixel 

flowchart TD;
    start-->id1(กำหนดจุดเริ่มต้น x1,y1);    
    id1-->id2(กำหนดจุดสิ้นสุด x2,y2);    
    id2-->id3(x = x1);
    id3-->id4{x>x2?};
    id4-- NO -->id5(คำนวณ y ตามสมการ 1.6);
    id5-->id6(DrawPixel บน coordinate x,y);
    id6-->id7(x = x+1);
    id7-->id4;
    id4-- YES -->id8(cleanup);
    id8-->stop;

เขียน code

  1. เปิดไฟล์ template.cpp แล้วบันทึกเป็น line.cpp

  2. แก้ไป title เป็น "Draw Line Segment"

    const char *title = "Draw Line Segment";

  3. เพิ่มชุดคำสั่งใน function draw() 

    void draw(){
/* TODO : draw a frame */
	int x1 = 0;
	int y1 = 0;
	int x2 = 500;
	int y2 = 500;
	int y;
	Color clr = WHITE;
	for(int x = x1;x<x2;x++){
		y = (int)(x * (y2 - y1)/(x2 - x1) + x2*y1 - x1*y2);
		DrawPixel(x,y,clr);
	}
}

  4. เพื่อให้เห็นภาพชัดขึ้นให้แก้ค่าสีพื้นหลังในบรรทัดที่ 40 จากเดิมกำหนดเป็น WHITE ให้เป็นสีเข้มขึ้นหรือดำ

    ClearBackground(BLACK);

  5. บันทึกแล้วทำการ build

การวาดหรือ plot ด้วย algorithm แบบนี้ดูเหมือนจะใช้ได้ แต่หากทดลองเปลียนค่าความชัน (ΔxΔy​) หลายๆค่า จะพบว่า ลักษณะของเส้นที่ปรากฏบางครั้งไม่ได้เป็นไปในแบบที่พอจะเรียกว่าเส้นตรง เช่น

  1. ความชันที่ทำให้เส้นตรงออกมาดูเป็นเส้นตรงจะอยู่ใกล้ๆกับ 1 ถ้าคิดเป็นค่าของมุมคือ 45 degrees หรือ 4π​ radians

  2. เมื่อความชันของเส้นตรงมีค่าน้อยๆ (m→0 )เส้นจะราบลงใกล้กับแนวนอน ภาพที่ออกมาจะมีลักษณะคล้ายกับบรรได เกิดการซ้ำของตัวเลขบนแกนนอนมากกว่าบนแกนตั้ง

  1. เมื่อความชันมีค่ามากๆ (m→∞ )จะพบว่ามีการกระโดดของ pixel เกิดการซ้ำของตัวเลขบนแกนตั้งมากกว่าบนแกนนอน

ดังที่กล่าวไว้ก่อนหน้าว่า pixel กับ point ไม่ใช่สิ่งเดียวกัน ตำแหน่งของ pixel ช่วยกำหนดบริเวณที่ point จะปรากฏบนหน้าจอได้ แต่ไม่ใช่ coordinates ของ point (บางท่านอาจเคยได้ยินประโยค If it seems right, it is right ) เพราะตัวเลข coordinates ของ point เป็นเลขจำนวนจริง ในขณะที่ตำแหน่งของ pixel เป็นจำนวนเต็ม การปรับแต่งตัวเลขจากจำนวนจริงไปเป็นจำนวนเต็มทำให้เกิดการเสียข้อมูลไปบางส่วน

สาเหตุเพระด้วยปัญหานี้จึงได้มีการพัฒนา algorithm ขึ้นมาเพื่อให้การวาดเส้นตรงดูเหมือนเส้นตรงมากขึ้น
